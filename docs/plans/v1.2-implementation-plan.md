## Validation Strategy

Since this is a plugin of markdown/JSON files (not compiled code), validation replaces traditional tests:

**Structural validation (run after every phase):**
```bash
# 1. All agents have required sections
for f in plugins/*/agents/*.md; do
  grep -q "^## Expertise" "$f" && \
  grep -q "^## Decision Framework" "$f" && \
  grep -q "^## Behavioral Directives" "$f" && \
  grep -q "^## Deliverables" "$f" || echo "FAIL: $f missing sections"
done

# 2. All skills have frontmatter description
for f in plugins/*/skills/*/SKILL.md; do
  head -5 "$f" | grep -q "^description:" || echo "FAIL: $f missing description"
done

# 3. No old squad names remain
grep -r "engineering-sprint\|campaign-ops\|sales-pipeline\|design-studio\|project-command\|power-tools" \
  --include="*.md" --include="*.json" plugins/ docs/ README.md CHANGELOG.md && echo "FAIL: old names found"

# 4. All JSON files are valid
for f in plugins/*/.claude-plugin/plugin.json plugins/*/hooks/hooks.json .claude-plugin/marketplace.json; do
  jq . "$f" > /dev/null 2>&1 || echo "FAIL: invalid JSON in $f"
done

# 5. All manifests are 1.2.0
for f in plugins/*/.claude-plugin/plugin.json; do
  jq -r '.version' "$f" | grep -q "1.2.0" || echo "FAIL: $f not bumped to 1.2.0"
done
```
